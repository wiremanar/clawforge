Module 2: AI-Powered Troubleshooting

The Prompt Formula

When you're stuck on a problem, use this template:

I'm troubleshooting a [SYSTEM TYPE] system.
Equipment: [SPECIFIC EQUIPMENT]
Symptom: [WHAT'S HAPPENING]
What I've already checked: [YOUR STEPS SO FAR]
Error codes: [ANY CODES]

What should I check next?

Example: VFD Fault

I'm troubleshooting an Allen-Bradley PowerFlex 525 VFD.
Symptom: Intermittent F33 (heatsink overtemp) fault, usually in afternoon
What I've already checked: Fan is running, ambient temp is ~85°F, load is ~70%
Error codes: F33, sometimes preceded by A51

What should I check next?

AI will suggest things like: check heatsink thermal paste, verify derating for ambient temp, check for blocked ventilation, inspect for dust buildup on heatsink fins. Not revolutionary on their own — but it's like having a second set of eyes instantly.

Pro Tips

  • Always verify AI suggestions against manufacturer documentation
  • Tell it your experience level — "I'm a journeyman electrician" gets better answers than nothing
  • Feed it error codes — AI is great at cross-referencing fault codes
  • Ask follow-up questions — "What's the most likely cause given the afternoon timing?" narrows it down

Module 3: Auto-Generate PLC Documentation

This is where AI saves you HOURS.

Documenting Existing Code

Copy a rung of ladder logic (or structured text) and paste it with this prompt:

Document this PLC code. For each rung/section:
1. What it does in plain English
2. What inputs/outputs are involved  
3. Any conditions or interlocks
4. Suggested rung comment

[PASTE YOUR CODE HERE]

Generating I/O Lists

I need an I/O list for a [SYSTEM DESCRIPTION].
Include: tag name, description, I/O type (DI/DO/AI/AO), wire number suggestion, typical signal type.
Format as a table.

Writing Functional Descriptions

Write a functional description for the following system:
[Describe your system]

Include: sequence of operation, interlocks, alarm conditions, HMI requirements.
Format it professionally — this goes to the customer.

The Money Move

Charge clients for documentation. Use AI to generate 80% of it in minutes. Spend your time reviewing and adding the specific details only you know. What used to take 8 hours now takes 2. Your rate stays the same. Your effective hourly rate just quadrupled.

Module 4: Prompt Engineering for Technical Work

The STAR Framework for Technical Prompts

  • Situation — What's the context?
  • Task — What do you need?
  • Additional Info — Specs, constraints, standards
  • Result Format — How do you want the output?

Example: Panel Layout Help

SITUATION: I'm designing a control panel for a water treatment pump station.
TASK: Help me create a component layout and suggest wire sizing.
ADDITIONAL: 3 x 15HP pumps (480V), Allen-Bradley CompactLogix, 120VAC control circuit, NEMA 4X enclosure, NEC 2023.
RESULT: Give me a component list with catalog numbers and a suggested layout description.

Power Moves

  • Ask AI to review your work: "Here's my control scheme. What am I missing?"
  • Cross-reference standards: "Does this panel design meet UL 508A requirements for [specific thing]?"
  • Generate submittals: AI can draft equipment submittals from spec sheets
  • Write proposals: Give it the scope, it drafts the proposal
