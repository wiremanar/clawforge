Module 1: What AI Agents Actually Are

The Simple Version

A chatbot waits for you to talk to it. An AI agent goes and does stuff.

Think of it like this:

  • Chatbot = A really smart person sitting at a desk, waiting for questions
  • AI Agent = That same person, but they have legs, arms, a phone, and a to-do list

What Makes an Agent Different

  1. Memory — It remembers previous conversations and context
  2. Tools — It can access the internet, read files, send emails, use APIs
  3. Autonomy — It can decide what to do next without you telling it every step
  4. Goals — You give it an objective, it figures out how to achieve it

Real-World Examples

  • An agent that monitors your email and drafts responses to common questions
  • An agent that researches competitors every Monday and sends you a summary
  • An agent that manages your social media posting schedule
  • An agent that qualifies leads and books meetings on your calendar
  • An agent that monitors prices and alerts you to deals

What Agents Can't Do (Yet)

  • Make genuinely creative leaps (they remix, they don't invent)
  • Handle truly novel situations with no precedent
  • Replace human judgment on important decisions
  • Work reliably without any human oversight

Module 3: Build Your First Agent (Step by Step)

Project: Email Auto-Responder Agent

We're going to build an agent that:

  1. Monitors a specific email label/folder
  2. Reads new emails
  3. Drafts appropriate responses
  4. Saves drafts for your review (NOT auto-sending — safety first)

Time required: 30-45 minutes Cost: $0 Platform: Make.com (free tier)

Step 1: Sign Up for Make.com

  1. Go to make.com
  2. Create a free account
  3. You get 1,000 operations per month — plenty for this

Step 2: Create a New Scenario

  1. Click "Create a new scenario"
  2. You'll see a blank canvas — this is where we build

Step 3: Add Gmail Trigger

  1. Click the "+" button
  2. Search for "Gmail"
  3. Select "Watch Emails"
  4. Connect your Gmail account (OAuth — safe, no password shared)
  5. Set filter: Choose a label (like "Needs Response") or leave as Inbox
  6. Set to check every 15 minutes

Step 4: Add AI Processing

  1. Click "+" after the Gmail module
  2. Search for "OpenAI" (or "HTTP" if you want to use Claude)
  3. Select "Create a Chat Completion"
  4. Connect your OpenAI API key (get one at platform.openai.com — free credits for new accounts)
  5. Set the prompt:
You are a professional email assistant for [YOUR BUSINESS NAME].

Read the following email and draft an appropriate response.

Rules:
- Be professional but friendly
- If it's a sales inquiry, express interest and suggest a call
- If it's a support question, answer if you can, or say you'll look into it
- If it's spam or irrelevant, respond with "SKIP"
- Keep responses under 150 words

Email subject: {{subject}}
Email from: {{from}}
Email body: {{body}}

Step 5: Add Draft Creation

  1. Click "+" after the AI module
  2. Add a "Filter" — only continue if AI response does NOT contain "SKIP"
  3. After the filter, add "Gmail — Create a Draft"
  4. Set "To" = original sender's email
  5. Set "Subject" = "Re: " + original subject
  6. Set "Body" = AI's response
  7. This creates a DRAFT — you review before sending

Step 6: Test & Activate

  1. Click "Run once" to test with your latest email
  2. Check your Gmail drafts — you should see a response ready to review
  3. If it works, click "Scheduling" → turn ON
  4. Set to run every 15 minutes

Congratulations — you just built your first AI agent. 🦾

Module 5: Agent Best Practices

The Golden Rules

1. Never let an agent send external communications without review

  • Always start with DRAFTS
  • Add a human approval step before anything goes to a customer
  • Build trust gradually — expand autonomy over time

2. Start simple, then layer

  • Your first agent should do ONE thing well
  • Add capabilities after it's proven reliable
  • Complex ≠ better

3. Monitor and iterate

  • Check your agent's output daily for the first week
  • Look for mistakes, weird responses, edge cases
  • Refine prompts based on what you see

4. Set boundaries

  • Rate limits (max emails per hour)
  • Content filters (never discuss X, Y, Z)
  • Fallback responses ("Let me check with the team and get back to you")

5. Keep costs in mind

  • Most AI API calls cost fractions of a cent
  • But high-volume agents can add up
  • Track your API usage: platform.openai.com/usage

Common Mistakes

  • ❌ Trying to automate everything at once
  • ❌ Giving agents access to sensitive accounts without limits
  • ❌ Not reviewing output before it goes to customers
  • ❌ Over-engineering prompts (simpler is usually better)
  • ❌ Forgetting to set error handling (what happens when the AI fails?)
